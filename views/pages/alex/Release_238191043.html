Alex Cook : Release

Created by Alex Cook on Mar 03, 2022

This is not a formal Scrum Event but Scrum does have suggestions and recommendations for Releases


“The way to delight [...clients…] is to either preempt or cultivate the desire for a feature, or be exceptionally responsive”

You do not need to release at the end of a sprint and therefore it is not a formal sprint event. An increment is not a prod release and should be viewable outside of production

Each product with have an ideal release frequency that reflects risk and cost of change

Release size should be weighed against the maximization of benefits and the reduction of waste

The smallest release size considered should be a new or improved outcome for a single persona

Releases and reviews help increase feedback so it is important for the Product Owner to weigh risk alongside the need for good feedback. While the Review allows us to get feedback from stakeholders, some areas for improvement may only show after repeated use of an increment.

Build

A build should be executed “Whenever new or modified code is pushed to the version control system.”
